# Impeccable Project Context

## Product
Browser game built around ML inference for manufacturing defect detection. Players reveal tiles on a build plate grid, attempting to expose safe zones (x1/x2 multiplier combos) while avoiding scrap tiles. The backend uses a real ML model (joblib) to classify zones. Grid sizes range from 3x3 to 11x11. Persistent high scores per configuration.

## Users
Manufacturing engineers and ML/data science students evaluating the model's real-world accuracy through gameplay. Also general tech enthusiasts drawn to the cyberpunk aesthetic. A portfolio piece demonstrating ML deployment.

## Visual Style
Cyberpunk industrial. Dark backgrounds with neon accent glow effects. Gritty, technical, and dramatic. Feels like a factory HUD crossed with a hacker game. Grid tiles should feel physical — satisfying to reveal.

## Colors
Primary: Cyan (#00E5FF) — safe tile reveals, active state, neon glow
Secondary: Violet (#7B2FBE) — UI chrome, headers, borders
Accent: Amber (#FFC107) — multiplier indicators, combo alerts, amber zones
Danger: Red (#FF1744) — scrap tiles, game over state, defect markers
Background: Zinc-950 (#0A0A0A) — near-black base
Text: Cyan-100 (#E0F7FA) on dark / Red-200 for warnings

## Typography
Monospace (JetBrains Mono or Courier) for all game UI — scores, coordinates, timer, grid labels. Creates a terminal/HUD aesthetic. Minimal body copy needed — this is a game.

## Layout & Spacing
Centered game canvas with control panel below/beside. Grid scales with tile count. Status bar at top (score, timer, combo multiplier). Controls panel for grid size, powder type, artifact toggle. Compact — all game elements visible without scrolling on desktop.

## Components
- Build plate tile grid (variable NxN, clickable)
- Tile reveal animation (safe / scrap / amber zone)
- Score + combo multiplier display
- Timer widget
- Grid size selector (3x3 to 11x11)
- Powder type toggle (Virgin / Recycled)
- Artifacts toggle
- High score display per config
- Animated loading screen with Render cold-start countdown
- Post-game full reveal overlay

## UX Principles
Tile reveals must be instant and satisfying — sub-100ms visual feedback. Scrap reveal should be dramatic (red flash, game-over state). Combo multipliers should feel rewarding with brief visual pop. Cold-start countdown manages expectations honestly. High scores are per-config — users should see their best for the current settings.

## Anti-patterns to Avoid
- Slow tile reveal animations that break game flow
- Confusing safe vs scrap visual distinction
- Hiding the multiplier mechanic without explanation
- Light/pastel UI that breaks the cyberpunk tone
- Generic Minesweeper UI cloned without personality

## Notes for AI Edits
Game logic hooks (useGame, useTimer) and backend API calls are off-limits. Tile grid layout is core gameplay — do not restructure. Safe to modify: visual styling, colors, glow effects, tile aesthetics, loading screen, score display design. Cyberpunk tone must be maintained — no softening of the visual direction.
